Why the 8 steps collapse into 5 agents

📦
Steps 1 + 2 → one agent. You can't decide which products without deciding where they go first. Portfolio selection and market sequencing are one decision, not two.
📈
Steps 3 + 4 → one agent. The forecast and the signal sensing loop are the same continuous function. Separating them creates artificial handoffs in what should be a live feedback loop.
🏭
Steps 5 + 6 → one agent. Planning a quantity without knowing where it's going is an incomplete optimization. Both steps solve the same problem with the same data.
⚠️
Step 7 → separate agent. Exception handling requires a distinct authority model — the agent reasons about whether it can act autonomously or must escalate. Different decision type.
🔁
Step 8 → separate agent. Learning is a meta-decision: what is worth keeping in the like-product library? Requires reasoning about knowledge quality, not just cycle outputs.
🧠
The original design's error: treating each process step as its own agent. That maps tasks to agents. True agent architecture maps decisions to agents — fewer boundaries, richer reasoning.
